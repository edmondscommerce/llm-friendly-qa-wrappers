# v0.1.0 - Initial Release

**Release Date**: 2026-02-17

## Summary

First release of QA Tool Wrappers with 14 wrappers across 4 languages. Each wrapper produces terse terminal output (1 line pass, 2-5 lines fail) and writes detailed JSON to a temp file. Native JSON output is used wherever tools support it.

## Highlights

- 14 QA tool wrappers across Node.js, PHP, Bash, and Python
- "Native JSON First" design principle - use tool's own JSON output when available
- Cross-language Python acceptance test harness
- Claude Code Hooks Daemon integration with plan workflow

## Wrappers

| Language | Tool | Native JSON? |
|----------|------|-------------|
| Node.js | ESLint | Yes (`--format json`) |
| Node.js | Prettier | No (custom) |
| Node.js | Jest | Yes (`--json`) |
| Node.js | TypeScript (tsc) | No (custom) |
| Node.js | Vitest | Yes (`--reporter=json`) |
| Node.js | Biome | Yes (`--reporter=json`) |
| PHP | PHPStan | Yes (`--error-format=json`) |
| PHP | PHP-CS-Fixer | Yes (`--format=json`) |
| PHP | PHPUnit | No (JUnit XML to JSON) |
| Bash | ShellCheck | Yes (`-f json`) |
| Bash | shfmt | No (diff to JSON) |
| Python | Ruff | Yes (`--output-format json`) |
| Python | pytest | No (custom) |
| Python | MyPy | Yes (`-O json`) |

## Design Principles

- **Native JSON First**: If a tool has native JSON output, use it directly
- **Terse Terminal Output**: 1 line on pass, 2-5 lines on fail
- **Tool-Specific Schemas**: Each tool defines its own JSON structure
- **Same-Language Wrappers**: Wrappers written in the tool's native language
- **Modern Versions Only**: Node >= 20, PHP >= 8.2, Python >= 3.11

## Installation

```bash
git clone -b v0.1.0 https://github.com/edmondscommerce/llm-friendly-qa-wrappers.git
cd llm-friendly-qa-wrappers

# Install per-wrapper dependencies as needed, e.g.:
cd wrappers/nodejs/eslint && npm install
cd wrappers/php/phpstan && composer install
pip install ruff pytest mypy
```
